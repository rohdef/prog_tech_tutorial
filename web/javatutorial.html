Programming techniques in Java - Tutorial

Another Java tutorial?

This torial actually focus more on programming techniques than Java. The resoning behind this tutorial is not to learn Java as such, although it will have that fortunate side effect. Here the focus is on programming techniques.

Java is chosen as the platform for multiple reasons:

  • Java is very strict, which makes it easier to do good code in less strict languages once good coding styles have been learned.
  • Types, since a lot of languages is dynamic, most people forget about types. Even those languages though use types and knowledge about them is definitely a help.
  • As a C style language the syntax is very similar to most languages you'd meet.
  • Java is a good choice for a future programming, as the platform is mature.
  • Android development is done in Java (desired by one of the original students).

More reasons could probably be found, but are really not necessary.

For the student

First a word of warning. This tutorial is designed to provide all you need. Do not search for help, etc. on Google. Do not use syntax, keywords etc. from other languages you may know, unless taught here. The tutorial is designed to teach you techniques more than Java. If you use outside sources, odds are that it will be counter productive. If techniques is not your goal, this will be the wrong tutorial for you.

The intended way of using this tutorial is that I start from the basics, so no fancy GUI etc., sorry. Please go with the flow, there's sanity behind the annoyance. All the graphical focus is nice, yes I do like it, but it is disturbing for the underlying basics. "You gotta learn how to walk before you can run." (said by Olivier Danvy a lot, my favorite teacher).

Also almost every step is followed by a lot of assigments. I will try to group them, so they mark a learning level. Practice makes best, so please solve all from each group to the level where you can do it in your sleep. I know, I know, it's a pain, yes! But trust me, you will be thankful for doing it. Just like everyone else. I will also try to provide hints, but try not to use them.

Stuck? Feel like you're stupid? Or perhaps that if you were just a little bit smarter, then...? Well first off, fear not. It is normal: Wikipedia on mathematical anxiety. Do not fear, rest, get some sleep and try again. Yes, I said rest :) Best advice ever, do no try to be clever when tired. Go to sleep and try again with a clear mind.

A word on IDE (Integrated Development Environment)

Most sane people do real world development in an IDE, such as IntelliJ or Eclipse. In an production situation these are definitely good choices, Emacs even better.

As this tutorial focus on techniques, please refrain from the temptation to use such a tool (Emacs without auto completion tools is fine :) ), as they will help you too much. For on Linux I recommend either Emacs, Vim, Kate or Gedit (last two is graphical) and on Windows Notepad++ will likely be good (haven't tested it).

If you use the VirtualBox image provided, things will be already prepared for you with a simple setup. See preparation instructions.

Summary

This is intended to teach you techniques as a primary goal. Please go with the flow. To get most benefit out of this, please follow these guidelines:

  • Do not search the net for help.
  • Avoid using the hints if possible
  • Don't panic! Get some rest, and don't try to learn when tired.
  • Do not use knowledge about syntax and keywords from other programming, what you need will be provided for.
  • Do not use overly helpful IDEs, learn the commands used and use a basic text editor

Provided this you should get a good and safe introduction to programming techniques and improve your skills rapidly (provided you don't already know them :p but even so, try to see if you really do, it may be educational).
