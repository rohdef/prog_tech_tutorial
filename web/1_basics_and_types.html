Let's get some code running. This assumes that you have set up Java and are familiar with basic console/terminal/command prompt (I will use terminal from here on) navigation - other tutorials for this will be added or linked to later. One of the reasons for doing it like this is to understand what an IDE (such as IntelliJ and Eclipse) actually does behind the scenes.

Write the code from above into a file (make sure the class name on line 1 matches the file name), consensus is to start with an upper case name and although Java allows lower case it's frowened upon and considered very bad style.

After the comment on line 3 (you can also delete the comment) add a new line, and add the following piece of code:

        System.out.println("Hello world!");

Now in the terminal type the following commands (change Bootstrap to your chosen name)

        javac Bootstrap.java
        java Bootstrap

If everything is done right your console should print out: "Hello world!". If not try to play around and see if you can get it to work. If you get entirely stuck there's a hint with the expected code, but try not to it.

Now that was awesome, right? Well OK, I guess I have to agree; that is a very round about way to get a computer to write something, but it's the beginning of something more. Now the code part is basically Java's command for Java to print something on the screen.

The next two is two terminal commands, first one is "javac" which is Java's compile command, this basically translates the code you wrote into something the computer understands. As you may already don't know how to just run what you wrote (not entirely true, there is interpreted languages) it has to translate it into something the machine understands - as Java is running on an VM there's technically more to it, but let's stick with the simplified explanation. The compilers task is to translate your code, which is something between human readable and machine code. What the compiler outputs is a .class- file corresponding to your file, this is your binary file - that is the one that can be run.

Lastly we have the command "java" which tells Java to run what's specified after it. That's basically it. Congratulations, you just wrote your first Java program. Now go ahead and do the following experiments (remember to do "javac" each time you change something, before running it):

  • Change the content in System.out.println, make sure to put it inside the quotes. Can you make it print your name?
  • Try to remove one of the quotes, read the error when you get. What could it mean?
  • Put the quote back there, and try to remove the semi colon at the end of the line. What does the error from "javac" mean? (Tip, remember this error, you will see it a lot).
  • Play around, put 3 prints in the code, try to provoke some errors from Java.

Types

Now the last boring (sorry but we need them foundations first) part. Good news is, if you are already used to static typed languages (eg. C, Pascal, C#) you can skip this part. Bad news if you ain't we need to cover it. So let's get it over with.

Understanding types is important, even in dynamic typed languages. A lot of people fight with things like JavaScript's coercion, and how languages behave, because they are not aware of the underlying type system. In dynamic languages you can typically write something like: var a = 42; and var a = '42'; without bothing that the first one is a number and the second one is a text. Often this is a nice feature, but there's a backside to it. If you are not aware what's going on it can lead to some strange errors.

In Java we operate with the following primitive data types:

  • byte a small integer of 8 bits (= 1 byte, surprise) that is values from -128 to 127. These aren't used that often. byte a = 42;
  • short a small posive integer of 16 bits, rarely used
  • int the normal integer type, it uses 32 bits. This is the most common of the integer types. int number = 5568421;
  • long a huge interger (64 bits) type used mostly for cases where int isn't enough, such as unix timestamps.
  • float is a small decimal point number, usually double is used today.
  • double a big decimal point number, the most common for decimal numbers today. double d = 3.1415926535;
  • boolean represents true and false values, mostly used for conditionals (code that states if something do something). bool b = true;
  • char a 16 bit unicode character. char c = 'q';

Also you have seen the non-primitive String, through a so called string literal, namely writing it between two quotes. A string can be defined like this: String s = "hey";. Note both the upper case S, in contrast to the primitive types and that it uses double quotes ", unlike char that uses single quotes '.

There's way more to types than written here. This is intended to get us started. Let's carry on an go to some fun and then play with conditionals and basic loops.
